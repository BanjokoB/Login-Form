Functional and non-functional requirements
Functional
The following are the identified functional requirements in the context of the brief:
• View products with transparent pricing and availability 
• Place orders for delivery or collection 
• Manage accounts and view order history 
• Sign up 
• Log in 
• Order management, scheduling and tracking 
• Loyalty and reward scheme
 • Accessibility features 
• Admin dashboard
Requirement	Priority 	Justification/ reason
View products with transparent pricing and availability	High	This would allow customers view the products they want to purchase.
Place orders for delivery or collection	High	This allows clients to select their orders and decide whether they want it delivered to an address of their choice or pickup from a collection point.
Manage accounts and view order history	Medium	This allows customers to view their accounts, change passwords, and view their order history, this is important but not as important as placing orders or viewing products pricing.
Sign-up	High	This feature allows customers to create an account.
Login	High 	This feature allows customers to login to their accounts to place and view orders.
Order management, scheduling and tracking	High	This feature allows users to manage and track their orders; it would also allow admins to schedule orders for the right delivery time.
Loyalty and reward scheme	High	This feature would allow customers to accumulate rewards and points based on how much they spend in the store. This would help increase customer retention. Although it is not clearly stated in the brief how the loyalty and reward scheme would work.
Accessibility features	High	This feature would include screen reader compatibility, high contrast mode, adjustable text size and clear navigation and more for customers with accessibility needs. In line with WCAG2.2 guidelines
Admin dashboard	Medium	This feature would allow admins to monitor orders, clients and stock levels.


Further breakdown of the functional requirements
Functional requirements are the features and operational requirements for the digital solution. Below are the functional requirements broken down into categories:
1.	User account management:
•	Users must be able to create an account 
•	Users must be able to login and logout securely
•	Users must be able to update personal information
•	Users must be able to view their order history
2.	Product browsing:
•	Customers must be able to view all available products
•	Products must display the product name, description, price, and availability 
•	Customers must be able to filter their search
3.	Order management:
•	Customers must be able to add items to basket
•	Customers must be able to place orders
•	Customers must be able to choose between collection or delivery
•	Customers must be able to select delivery or collection time 
•	Customers must be able to track order status
4.	Loyalty scheme and rewards:
•	Customers must be able to earn points on purchases
•	Customers must be able to redeem points for discounts
5.	Staff dashboard:
•	Staff must be able to access and view dashboards
•	Staff must be able to add new products
•	Staff must be able to update product information
•	Staff must be able to manage and view stock levels
•	Staff must be able to view orders 
Non-functional
The following are the non-functional requirements in the context of the brief: 
•	System performance
•	Security
•	Accessibility
•	Reliability
•	Scalability
•	Compatibility
•	Maintainability 
•	Data backup and recovery
Requirement	Priority	Justification/reason
System performance
	High	The system should load pages within a few second (ideally <3) so users can browse quickly and place orders without delay.
Security 	High	The system must protect user data such as personal information or bank details using encryption and secure authentication to prevent unauthorized access in line with GDPR. 
Accessibility	High	The system must include screen reader compatibility, high contrast mode, adjustable text size and clear navigation and more for customers with accessibility needs. In line with WCAG2.2 guidelines.
Scalability	Medium	The system should be able to handle growth in the number of users, products and staff/admins as the company expands in the future.
Compatibility	Medium	The system should work across multiple devices and browsers to ensure all users can access the system.
Maintainability	Medium 	The system should be designed in a way that would allow developers fix bugs, update the system and add new features without affecting the current functionality or needing the original developer.
Data backup and recovery	High	The system should regularly back up data such as orders, accounts and product information in case of a system failure.





Further breakdown of the non-functional requirements	
Non-functional requirements refer to the system quality, performance and constraints. Below are the non-functional requirements broken down further:
1.	Performance:
•	The system must load pages in under 3 seconds.
•	The system must support at least 500 concurrent users
2.	Security:
•	User passwords must be encrypted and securely stored
•	The system must comply with data protection regulations (GDPR)
•	The system must use secure authentication
3.	Accessibility:
•	The platform must comply with WCAG2.2 standards
•	The system must support/use screen readers 
•	The system must allow users navigate with just the keyboard
4.	Usability:
•	The user interface must be easy to see and navigate
•	Product information must be clearly presented
•	The system must support all devices (mobile,desktop,tab)
5.	Reliability:
•	The system must have 99% uptime
•	The system must backup data daily
6.	Scalability: 
•	The system must allow new products and information to be added easily
•	The system must be able to handle future growth in users and orders
7.	Maintainability: 
•	Code must follow best industry practices (PEP-8 etc)
•	The system must be easy for developers to update
8.	Compatibility:
•	The system must work on modern browsers.
