Functional and non-functional requirements 
Functional 
The following are the functional requirements identified from the context of the brief. 
•	View products with transparent pricing and availability
•	Place orders for delivery or collection
•	Manage accounts and view order history
•	Sign up
•	Log in 
•	Order management, scheduling and tracking
•	Loyalty and reward scheme 
•	Accessibility features
•	Admin dashboard
Requirements	Priority 	Justification/reason
View products with transparent pricing and availability 	High	This would allow customers view the products they want to purchase. The information about price would allow customers.
Place orders for collection or delivery	High	This allows clients to select their orders and decide whether they want it delivered to an address of their choice or pickup from a pickup point.
Manage accounts and view order history	Medium	This allows customers to view their accounts, change passwords, and view their order history, this is important but not as important as placing orders or viewing products pricing.
Sign-up	High	This feature allows customers to create an account.
Login	High	This feature allows customers to login to their accounts to place and view orders.
Order management, scheduling and tracking
	High	This feature allows users to manage and track their orders; it would also allow admins to schedule orders for the right delivery time.
Loyalty and reward scheme	High	This feature would allow customers to accumulate rewards and points based on how much they spend on the website. Although it is not clearly stated on the brief how the scheme would work.
Accessibility features 	High 	This feature would include screen reader compatibility, high contrast mode, adjustable text size and clear navigation and more for customers with accessibility needs.
Admin dashboard	Medium	This feature would allow admins to monitor orders, clients and stock levels.
